\chapter{Clinical Decision Support Dashboard}

\section{System Architecture}

To translate the trained machine learning models into a practical tool for clinical application, a production-grade full-stack web application was developed. The architecture operates entirely as an interactive clinical decision support dashboard, composed of a decoupled backend API and a dynamic frontend user interface.

\subsection{Backend Inference Engine (FastAPI)}
The backend is authored in Python utilizing the FastAPI framework to expose the serialized machine learning models via RESTful endpoints. Its core responsibilities include:
\begin{itemize}
    \item \textbf{Model Instantiation:} Pre-loading the best-performing serialized \texttt{.pkl} algorithms (e.g., Random Forest for Diameter, Gradient Boosting for Length) into server memory upon startup to eliminate disk I/O bottlenecks during live inference.
    \item \textbf{Data Validation:} Enforcing strict schema evaluation of incoming clinical parameters (Power, Time, Antenna Type) using Pydantic, ensuring that out-of-bound or corrupted inputs are rejected before reaching the predictive algorithms.
    \item \textbf{Ensemble Prediction:} Routing validated parameters through the appropriate feature engineering pathways and returning both the predicted ablation zone dimensions (effective diameter and length) alongside derived metrics such as estimated volume and sphericity index.
\end{itemize}

\subsection{Frontend User Interface (React)}
The user interface is engineered as a modern Single Page Application (SPA) using React and Vite. It is designed specifically for clinical environments with a dark-mode aesthetic to minimize visual fatigue. Key features of the frontend include:
\begin{itemize}
    \item \textbf{Interactive Input Form:} Clinicians can rapidly input power settings, duration, and select specific antenna types from a comprehensive drop-down menu mapped exactly to the trained categorical variables.
    \item \textbf{Real-Time KPI Dashboard:} As processing concludes, the system populates massive Key Performance Indicator (KPI) cards displaying the projected ablation focus diameter, total length, estimated ablation volume, and sphericity. 
    \item \textbf{Algorithmic Transparency:} The dashboard deliberately exposes the underlying algorithm metadata utilized for the prediction (including the specific model name and its cross-validated median absolute error) to build clinician trust in the AI-generated results.
\end{itemize}
